Tentang Aplikasi

Aplikasi pemantauan kualitas air berbasis Fuzzy Sugeno dengan sensor pH, kekeruhan (NTU), dan suhu.

v1.0.0 / 2025-10
Metode yang Digunakan

Sistem inferensi Sugeno orde-0 digunakan untuk menilai kualitas air berdasarkan input sensor. Proses ini melibatkan tiga tahap utama: fuzzifikasi, inferensi aturan fuzzy, dan defuzzifikasi.

  • Fuzzifikasi: Nilai pH, NTU, dan suhu diubah menjadi derajat keanggotaan (membership degree).
  • Inferensi: Setiap aturan fuzzy menghasilkan bobot (wi) berdasarkan operator logika fuzzy.
  • Defuzzifikasi: Nilai keluaran dihitung dengan rata-rata berbobot (weighted average) seperti berikut:

score = Σi(wi·ci) / Σiwi

Di mana:

  • wi = derajat keanggotaan aturan ke-i
  • ci = konstanta keluaran (nilai crisp) dari aturan ke-i
  • score = hasil defuzzifikasi berupa skor 0–100

Bobot wi diperoleh dari operator logika fuzzy: produk (AND), max/min (OR). Metode ini cocok untuk sistem real-time karena perhitungannya ringan namun tetap presisi.

  • Input: pH (0–14), NTU (0–1000), Suhu (0–40°C)
  • Output: Skor 0–100 + label Excellent / Good / Fair / Poor

Ringkasan Aturan Fuzzy
  • R1: pH Netral ∧ NTU Jernih ∧ Suhu Nyaman → c = 95
  • R2: pH Netral ∧ NTU Jernih → c = 85
  • R3/R4: Kombinasi Netral/Nyaman/Jernih → c = 80
  • R5–R8: Sedang/basa/dingin/panas → c = 50–60
  • R9–R11: Keruh/ekstrem → c = 20–25

Setiap aturan di atas mewakili kondisi lingkungan air tertentu. Nilai c adalah konsekuen (output crisp) yang akan digunakan pada proses defuzzifikasi untuk menghasilkan skor akhir.

Contoh: Jika sensor mendeteksi pH = 7.2 (Netral), NTU = 10 (Jernih), dan Suhu = 28°C (Nyaman), maka aturan R1 aktif paling kuat, menghasilkan skor sekitar 95 dengan label Excellent.

Kalibrasi & Best Practice
  1. Mulai dari MF default di halaman Kalibrasi MF.
  2. Bandingkan hasil skor dengan uji lab/acuan (mis. Permenkes/WHO).
  3. Geser titik trap/tri sedikit demi sedikit, cek dampaknya pada data sampel.
  4. Validasi pada rentang kondisi yang berbeda (jernih → keruh, dingin → panas).
  5. Dokumentasikan versi konfigurasi MF yang dipakai.
Sesuaikan juga pemetaan label (ambang skor) dengan kebutuhan operasional.
FAQ

Metode logika fuzzy dengan konsekuen berupa konstanta/linear. Di sini dipakai orde-0 (konstanta). Nilai akhir didapat dari rata-rata berbobot (weighted average).

Bisa. Ganti sumber data di halaman Input/Dashboard atau integrasikan API/serial port, lalu gunakan fungsi perhitungan yang sama untuk skornya.

Tidak wajib. Ambang label (Excellent/Good/Fair/Poor) bisa disesuaikan untuk konteks dan regulasi yang kamu pakai.
